# Contributing

感谢你愿意参与 Forensic Suite Toolkit。这个项目希望把 Windows 虚拟机打造成更适合数字取证、应急响应、CTF Forensics 和安全分析工作的工具环境。

## 可以贡献什么

- 推荐新的取证工具、应急工具、流量分析工具、内存分析工具、移动端取证工具。
- 修正文档中的工具名称、版本、链接、分类和使用说明。
- 提交安装脚本、配置脚本、快捷启动方式或镜像构建建议。
- 补充截图、案例、工具使用注意事项和常见问题。
- 帮忙验证工具是否仍然可下载、可安装、可正常运行。

## 推荐工具时请尽量提供

- 工具名称。
- 官方网站或 GitHub/GitLab 仓库地址。
- 许可证信息。
- 适用场景，例如磁盘取证、内存取证、流量分析、日志分析、移动端取证、恶意样本分析、CTF Forensics。
- 推荐版本。
- 是否支持 Windows、WSL、Kali 或便携运行。
- 是否需要联网、账号、商业授权或额外驱动。

## 收录原则

- 优先收录来源清晰、许可证明确、仍在维护的工具。
- 优先选择官方发布渠道或可信构建产物。
- 不收录明显侵权、来历不明、夹带后门或无法验证来源的二进制文件。
- 商业软件只记录官方入口和使用说明，不提交未授权破解文件。
- 高风险工具需要写明合法授权使用边界。

## Pull Request 建议

- 一次 PR 尽量只处理一个明确主题。
- 文档修改请保持分类清晰，避免重复工具。
- 新增脚本请写明运行环境、依赖和回滚方式。
- 如果引入第三方资源，请保留原作者、来源链接和许可证信息。

## 行为准则

请保持讨论专业、友善、可复现。不同取证流程和工具偏好可以并存，项目更看重清晰来源、稳定使用和可持续维护。
